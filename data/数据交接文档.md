# 仓库与队伍物资数据交接文档

本文档面向数据交接与二次分析使用者，系统性说明“仓库与队伍的物资数据”的来源、结构、字段、关系、数据量、数据类型/格式、质量特征、分析口径与注意事项，保证交接方能在不依赖上下文代码的前提下理解并正确使用数据。

## 1. 数据范围与位置

- 源数据目录：`仓库和队伍的物资数据`
  - `em_material.xls` — 物资主表（Material）
  - `em_warehouse.xls` — 仓库主表（Warehouse）
  - `em_warehouse_material.xls` — 仓库-物资关联表（Warehouse-Material M2M）
  - `rescue_team.xls` — 救援队伍主表（Rescue Team）
  - `rescue_team_material.xls` — 队伍-物资关联表（Team-Material M2M）

> 注：文件为 Excel 97-2003 `.xls` 格式，所有表的首个工作表为有效数据表，首行是字段名。

## 2. 数据量与完整性概览

- 物资（em_material）：10983 条
- 仓库（em_warehouse）：302 条（源表 303 行；存在 1 条异常/空 id 记录）
- 队伍（rescue_team）：238 条（源表 239 行；存在 1 条异常/空 id 记录）
- 仓库-物资映射（em_warehouse_material）：8261 对，覆盖 246 个仓库
- 队伍-物资映射（rescue_team_material）：521 对，覆盖 150 支队伍
- 完整性检查（基于 id 匹配）：映射表中的 `warehouse_id`、`team_id`、`material_id` 均能在对应主表找到匹配（未发现映射“孤儿”id）。

## 3. 数据模型与关系（ER 概要）

- 实体与主键：
  - Material(id)
  - Warehouse(id)
  - Team(id)
- 关系（多对多）：
  - Warehouse(id) ←→ WarehouseMaterial(warehouse_id, material_id) ←→ Material(id)
  - Team(id) ←→ TeamMaterial(team_id, material_id) ←→ Material(id)
- 约束建议（逻辑约束）：
  - `em_warehouse_material.warehouse_id ∈ em_warehouse.id`
  - `em_warehouse_material.material_id ∈ em_material.id`
  - `rescue_team_material.team_id ∈ rescue_team.id`
  - `rescue_team_material.material_id ∈ em_material.id`

> 说明：`em_material` 表内存在 `team_id`、`warehouse_id` 字段，但从全量关系表达与数据覆盖角度，建议以映射表为主建立关联，不直接依赖物资表内的外键列。

## 4. 各表字段说明（字段名按表头）

### 4.1 物资表 em_material.xls（19 列）

- 主键
  - `id`：物资条目唯一标识（字符串）。少量记录存在异常（如 `#NUM!` 或空）。
- 核心描述
  - `material_name`：物资名称（如“救生圈”“剪刀”“限速30标志”）。
  - `material_type`：物资类别代码/枚举（常见值 0/1；未提供字典，建议作为枚举处理）。
  - `spec_model`：规格型号。
  - `quantity`：数量（数值型，整数或小数）。
  - `unit`：单位（如“个/把/块/套”等）。
  - `status`：状态标记（0/1 数值）。
  - `expiry_date`：到期时间（Excel 日期；详见“数据类型与格式”）。
- 管理与溯源
  - `remark`：备注。
  - `create_time`，`update_time`：创建/更新时间（Excel 日期/时间）。
  - `creator`，`updater`：创建/更新人。
  - `del_flag`：删除标记（0/1 数值）。
- 所属与区域
  - `team_id`，`warehouse_id`：关联队伍/仓库（不保证全量覆盖，推荐使用映射表）。
  - `administrative_divisions_id`，`administrative_divisions_name`：行政区划信息。
  - `stars`：评分/星级（语义需结合业务字典）。

### 4.2 仓库表 em_warehouse.xls（24 列）

- 主键

  - `stake`：桩号/里程标记（如 `K103+496`）。
  - `latitude`，`longitude`：坐标（浮点数，经纬度）。
- 管理与校验
  - `remark`：备注。
  - `create_time`，`update_time`：创建/更新时间（Excel 日期/时间）。
  - `creator`，`updater`：创建/更新人。
  - `del_flag`：删除标记（0/1 数值）。

  - `unitor`，`unitor_id`，`unitor_tel`：管养单位信息。

### 4.3 仓库-物资映射表 em_warehouse_material.xls（2 列）

## 5. 数据类型与格式（统一口径）

- 日期/时间字段（如 `create_time`、`update_time`、`expiry_date`、`last_verified_at`、`next_due_at`）在 Excel 中多以“日期序列数”存储（1900 系统）。二次处理时建议：
  - 将序列数转换为 ISO 8601 文本，如 `2025-08-22T21:43:27`。
  - 对看似日期的数值（大致 20000–60000）尝试转换；失败时保留原值并记日志。
- 数值字段（如 `quantity`、`team_size`）：
  - 保留整数外观（3.0 → 3），其余为浮点。
- 电话/联系方式字段（如 `contact_phone`、`leader_phone`、`unitor_tel`）：
  - 建议读取为字符串，去掉尾随 `.0`，保留全量字符（避免丢失前导 0）。
- 布尔/状态字段（如 `status`、`del_flag`、`verification_state`）：
  - 源中以 0/1 数值表达，建议在建模时转为布尔或显式枚举（并保留原始值）。
- 空值：
  - 空单元格读取为 `null`（或空字符串，视目标系统策略）。

## 6. 数据质量与风险点

- 主键异常：
  - 仓库与队伍主表各有 1 条记录 `id` 为空/异常（相较总行数少 1）。建议导入数据库时跳过或回补合法 id。
  - 物资表中观测到 `id` 异常案例（`#NUM!`），建议清洗时过滤或修复。
- 关系覆盖：
  - 仓库-物资映射覆盖 246/302 个仓库；队伍-物资映射覆盖 150/238 支队伍。统计时区分“有映射/无映射”实体。
- 口径一致性：
  - 物资数据更接近“物资条目级”（含 `quantity`、`unit`），而非“品类级”。如需品类口径统计，务必基于 `material_name + spec_model + unit` 聚合后再统计数量或件数。
- 字典缺失：
  - `material_type`、`warehouse_type`、`team_type` 等枚举无字典；分析中需避免主观映射，或在有权威字典后回填。
- 坐标与桩号：
  - `latitude/longitude` 可用于地图展示；`stake` 为文本桩号（如 `K103+496`），若需空间定位需自定义解析规则并结合道路中心线数据。

## 7. 使用与分析建议（示例口径）

- 某仓库的物资清单：
  1) 在 `em_warehouse_material` 取目标 `warehouse_id` 的全部 `material_id`；
  2) 在 `em_material` 以 `id ∈ material_id` 过滤。

- 某队伍的物资清单：
  1) 在 `rescue_team_material` 取 `team_id` 对应的 `material_id`；
  2) 在 `em_material` 以 `id ∈ material_id` 过滤。

- 品类级库存统计（示例逻辑）：
  - 以 `material_name + spec_model + unit` 作为品类键聚合 `quantity`，保留 `status` 为过滤条件（如仅统计未删除、在库状态）。

- 行政区/单位维度统计：
  - 物资：可用 `administrative_divisions_id/name` 汇总；
  - 仓库/队伍：可用 `belong_org_*`、`unitor_*` 或结合外部机构字典归并。

- 地图展示：
  - 以仓库/队伍的 `latitude/longitude` 绘制点要素，点击气泡展示关联物资条目数与负责人、联系方式等基础信息。

- 简要 SQL 片段（示意）：
  - 某仓库物资条目数：
    ```sql
    SELECT COUNT(*)
    FROM em_warehouse_material wm
    WHERE wm.warehouse_id = :warehouse_id;
    ```
  - 某队伍关联物资详情：
    ```sql
    SELECT m.*
    FROM rescue_team_material tm
    JOIN em_material m ON m.id = tm.material_id
    WHERE tm.team_id = :team_id;
    ```
  - 物资品类聚合：
    ```sql
    SELECT material_name, spec_model, unit, SUM(quantity) AS total_qty
    FROM em_material
    WHERE COALESCE(del_flag, 0) = 0
    GROUP BY material_name, spec_model, unit;
    ```

## 8. 合规与权限

- 数据包含姓名与电话等个人联系信息，仅限内部使用。对外展示或分享前需脱敏（去除手机号或仅显示后四位）。
- 如涉及地图展示，注意不泄露敏感设施的精确位置与联系方式。

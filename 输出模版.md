# 应急指挥方案标准输出格式规范

> 定位：规定模型最终输出的应急指挥方案必须遵循的结构和内容要求
> 版本：v1.0

---

## 1. 设计原则

- **有依据**：每个关键决策标注预案来源或案例依据
- **可执行**：步骤具体到责任单位、行动内容、时间要求
- **符合规范**：遵循应急预案的标准章节结构，指挥员拿到就能用
- **格式固定**：每次输出严格按相同结构，不随模型发挥而变化

---

## 2. 方案标准结构

方案共分为 **9 个固定章节**，模型每次必须按顺序输出：

```
一、事件概述
二、响应定级
三、指挥架构
四、预警发布
五、处置行动方案
六、资源调度方案
七、信息报送与新闻发布
八、风险提示与注意事项
九、依据引用
```

---

## 3. 各章节内容规范

### 一、事件概述

**作用**：让指挥员 10 秒内掌握事件全貌。

**必填字段**：

| 字段 | 说明 | 示例 |
|------|------|------|
| 事件类型 | 事故分类 | 交通事故（多车追尾） |
| 事发时间 | 精确到分钟 | 2026年4月12日 14:30 |
| 事发位置 | 路段+桩号或具体地址 | G72高速K85公里处（桂林市临桂区段） |
| 经纬度 | 如已获取 | 25.2341°N, 110.1823°E |
| 事件描述 | 一句话概括 | 因暴雨导致山体滑坡，双向道路中断，2辆车被埋，2人被困 |
| 伤亡情况 | 死亡/受伤/被困/失联 | 被困2人，暂无死亡报告 |
| 道路影响 | 中断/拥堵范围 | 双向交通中断，后方积压车辆约3公里 |
| 天气状况 | 当前天气 | 暴雨，能见度低，风力4-5级 |
| 路况状况 | 周边路况 | 周边G321国道通行正常，可作为绕行路线 |

**格式要求**：表格形式呈现，简洁明了。

---

### 二、响应定级

**作用**：明确本次事件的响应级别、由谁启动响应、适用什么预案。

**必填内容**：

| 字段 | 说明 | 示例 |
|------|------|------|
| 响应级别 | I/II/III/IV级 | III级（较大） |
| 定级依据 | 对照预案分级标准的哪一条 | 造成3人以上10人以下死亡或失踪 |
| 响应启动主体 | 由谁启动 | 事发地设区市人民政府启动应急响应 |
| 适用预案 | 本次参考的预案名称 | 《广西高速公路突发事件应急预案》 |
| 叠加预案 | 如涉及灾害类别 | 《广西交通运输防洪防台应急预案》 |
| 预案依据原文 | 引用分级标准的原文 | "造成高速公路毁坏、双幅交通中断...抢修处置时间预计在6小时以上"——《广西高速公路突发事件应急预案》7.2节 |

**格式要求**：明确写出定级理由，不能只写"III级"而不解释为什么。

---

### 三、指挥架构

**作用**：明确谁来指挥、设哪些工作组、各组负责什么。

**必填内容**：

```
总指挥：XXX（职务）
副总指挥：XXX（职务）

工作组设置：
┌──────────────────┬───────────────┬──────────────────────────┐
│ 工作组           │ 牵头单位       │ 主要职责                   │
├──────────────────┼───────────────┼──────────────────────────┤
│ 人员搜救工作组   │ 消防救援部门   │ 现场人员搜救、被困人员营救   │
│ 交通疏导工作组   │ 公安交警部门   │ 交通管制、车辆分流疏导       │
│ 医疗救援工作组   │ 卫生健康部门   │ 伤员现场救治和转运           │
│ 抢修抢险工作组   │ 高速公路经营者 │ 道路抢通、现场清理           │
│ ...              │ ...           │ ...                        │
└──────────────────┴───────────────┴──────────────────────────┘

依据：《广西高速公路突发事件应急预案》第2节、第4.2.1节
```

**格式要求**：表格形式，每个工作组必须有牵头单位和职责，末尾标注预案来源。

---

### 四、预警发布

**作用**：明确应发布什么级别的预警、谁发布、通过什么渠道。

**必填内容**：

| 字段 | 内容 |
|------|------|
| 预警级别 | 黄色预警（III级） |
| 发布主体 | 事发地设区市人民政府 |
| 发布流程 | 属地县级人民政府报设区市人民政府，由设区市人民政府向社会公众发布 |
| 发布渠道 | 收费站可变信息板、交通广播、政府网站、新媒体 |
| 预警内容要点 | G72高速K85处因山体滑坡双向中断，请过往车辆绕行G321国道 |
| 预案依据 | 《广西高速公路突发事件应急预案》3.2节 |

**如适用**，附上预警通知模板（填充后的版本）。

---

### 五、处置行动方案

**作用**：这是方案的核心部分——具体怎么干。

**结构要求**：按时间阶段分为三个阶段，每个阶段列出具体行动项。

#### 第一阶段：先期处置（事发后0-30分钟）

| 序号 | 行动内容 | 责任单位 | 时间要求 | 预案依据 |
|------|----------|----------|----------|----------|
| 1 | 封锁事故现场，设置警戒区域 | 公安交警部门 | 立即 | 预案4.2.3节(3) |
| 2 | 在事故上游1km处设置警示标志和分流点 | 高速公路经营管理者 | 15分钟内 | 预案4.2.3节(1) |
| 3 | 组织消防力量开展被困人员搜救 | 消防救援部门 | 接警后立即出动 | 预案4.2.1节(1) |
| 4 | 调派救护车赶赴现场 | 卫生健康部门 | 接报后15分钟内出发 | 预案4.2.1节(5) |
| 5 | 启动交通分流方案，引导车辆从最近出口下高速 | 公安交警+高速管理方 | 20分钟内 | 预案4.2.1节(4) |

#### 第二阶段：全面响应（30分钟-2小时）

| 序号 | 行动内容 | 责任单位 | 时间要求 | 预案依据 |
|------|----------|----------|----------|----------|
| 6 | 重伤员转运至最近有条件的医疗机构 | 医疗救援工作组 | 视伤情 | 预案4.2.1节(5) |
| 7 | 调动排障设备开展现场清理 | 抢修抢险工作组 | 到达现场后立即 | 预案4.2.1节(8) |
| 8 | 发布绕行方案，引导远端车辆改道 | 交通疏导工作组 | 1小时内 | 预案4.2.1节(4) |
| 9 | 监测滑坡体稳定性，评估二次滑坡风险 | 专家技术支持组 | 到达现场后 | 预案2.5节 |
| 10 | 滞留司乘人员接驳转运 | 司乘转运工作组 | 视滞留规模 | 预案4.2.1节(3) |

#### 第三阶段：持续处置与恢复（2小时以后）

| 序号 | 行动内容 | 责任单位 | 时间要求 | 预案依据 |
|------|----------|----------|----------|----------|
| 11 | 按"先通后善"原则抢通道路 | 高速公路经营管理者 | 视损毁程度 | 预案4.2.3节(1) |
| 12 | 确认道路具备通行条件后恢复通行 | 公安交警+高速管理方 | 经检测确认后 | 预案5.3节 |
| 13 | 现场全面清理和路产设施修复 | 高速公路经营管理者 | 通行恢复后持续 | 预案4.2.1节(7) |

**格式要求**：
- 每个行动项必须有明确的责任单位和时间要求
- 预案依据列标注具体章节号
- 行动内容要具体，避免"做好现场管控"这种空泛表述

### 七、信息报送与新闻发布

**作用**：明确信息报给谁、多久报、怎么发新闻。

#### 信息报送

| 报送事项 | 时限 | 报送对象 | 方式 |
|----------|------|----------|------|
| 事件初报 | 事发后15分钟内 | 高速公路管理机构、公安交警部门 | 电话 |

| 情况续报 | 有新进展时 | 上级应急指挥机构 | 电话+书面 |

#### 新闻发布

| 事项 | 内容 |
|------|------|
| 发布主体 | 事发地人民政府或授权的交通运输部门 |

| 发布渠道 | 交通广播、政府网站、新媒体、收费站信息板 |

依据：《广西高速公路突发事件应急预案》4.2.1节(10)

### 八、风险提示与注意事项

**作用**：提醒指挥员关注可能的风险和特殊注意事项。

**内容分为三类**：

#### 安全风险
- 暴雨仍在持续，存在二次滑坡风险，救援人员注意自身安全防护
- 事故现场可能有车辆燃油泄漏，注意防火

#### 处置风险
- 救援路线经过拥堵路段（G72 K80-K83），第一梯队可能延迟5-10分钟
- 夜间作业能见度低，需确保照明设备到位

#### 衍生风险
- 双向中断可能导致后方车辆持续积压，需提前启动远端分流
- 滞留时间过长可能引发司乘人员情绪问题，需做好安抚工作

## 4. 输出格式要求

### 4.1 强制规则

1. **9个章节必须完整输出**，不能遗漏任何章节
2. **章节顺序固定**，不能调换
3. **每个行动项必须标注预案依据**，写到具体章节号
4. **责任单位必须具体**，不能写"相关部门"
5. **时间要求必须量化**，不能写"尽快"
6. **资源调度必须有联系人和电话**

### 4.2 信息不足时的处理

当某些信息尚未获取时，该字段标注 `[待确认]` 并说明缺什么：

```
伤亡情况：被困2人，死亡人数[待确认——已向现场询问，等待回复]
```

### 4.3 方案版本管理

方案输出时标注版本号和生成时间：

```
本方案为第1版，生成时间：2026-04-12 14:45
如有信息更新，将生成修订版本
```

## 5. 在 Skill 中的落地方式

这份输出格式规范应作为**主流程 Skill 的 prompt 的一部分**，在 OUTPUT 阶段注入。

具体做法：在 master_flow 的 prompt.md 中增加以下指导：

```
## OUTPUT 阶段输出规范

你现在需要输出最终的应急指挥方案。请严格按照以下9个章节顺序输出，
不得遗漏或调换：

一、事件概述（表格形式，10秒内让指挥员掌握全貌）
二、响应定级（级别+定级依据+预案原文引用）
三、指挥架构（表格：工作组/牵头单位/职责）
四、预警发布（级别/发布主体/渠道/内容要点）
五、处置行动方案（按三个阶段分组，每项有责任单位+时间+预案依据）
六、资源调度方案（按梯队分组，含联系人电话）
七、信息报送与新闻发布（报送时限+对象+新闻发布安排）
八、风险提示与注意事项（安全风险/处置风险/衍生风险）
九、依据引用（汇总所有引用的预案章节和历史案例）

关键规则：
- 每个行动步骤必须标注预案依据（如"预案4.2.1节(3)"）
- 责任单位必须具体，不能写"相关部门"
- 时间要求必须量化，不能写"尽快"
- 信息不足时标注[待确认]
```
